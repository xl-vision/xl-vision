## 方法

组件提供了一些静态方法，使用方式和参数如下：

- `Message.success(config | string)`
- `Message.error(config | string)`
- `Message.info(config | string)`
- `Message.warning(config | string)`
- `Message.loading(config | string)`
- `Message.open(config | string)`

`config` 类型如下

| 参数          | 说明                                          | 类型       | 默认值 | 是否必填 |
| ------------- | --------------------------------------------- | ---------- | ------ | -------- |
| content       | 提示内容                                      | ReactNode  | -      | true     |
| duration      | 自动关闭的延时，单位毫秒。设为 0 时不自动关闭 | number     | 3000   | -        |
| icon          | 自定义图标                                    | ReactNode  | -      | -        |
| closeIcon     | 自定义关闭图标                                | ReactNode  | -      | -        |
| showClose     | 是否展示关闭图标                              | boolean    | -      | -        |
| onAfterClosed | `Message`关闭后回调                           | () => void | -      | -        |

> 除上述参数外，也支持基本的 `HTMLAttributes`，如`className`、`style`。

### setGlobalConfig

> 对于全局方法，无法通过上下文获取`ThemeContext`，因此我们也提供了手动设置的方法

```ts
Message.setGlobalConfig({
  themeProviderProps:  Omit<ThemeProviderProps, 'children'>,
  top?: number;
  container?: PortalContainerType<HTMLElement>;
});
```

具体可参考[ThemeProvider](/components/ThemeProvider)。
